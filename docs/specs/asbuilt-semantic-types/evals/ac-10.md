# Eval AC10 — reruns are free and identical

Observable success: Running reclassify twice with the same inputs yields
byte-identical bundles, and the second run reports zero applied — an
operator can rerun the backfill anytime without diffs appearing. No clock
reads, no randomness anywhere in the applier.

Anti-patterns this catches: nondeterminism in a mechanical applier (the
asbuilt suite's load-bearing invariant — timestamps, ordering instability,
randomness); non-idempotent backfill that makes reruns scary.

Would fail if:
- Two identical runs differ by any byte in any written file.
- The second run applies anything (idempotence broken).
- Output ordering depends on filesystem enumeration order rather than a
  codepoint sort.
